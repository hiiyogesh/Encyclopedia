# 📚 Encyclopedia — Local AI RAG Chatbot

A **fully local, privacy-first** AI assistant that lets you upload PDF or JSON documents and ask questions about them. All language model inference and embedding generation run **on your own machine** via [Ollama](https://ollama.ai), while document vectors are stored in your own private [Supabase](https://supabase.com) instance.

> **Internal Use Only · Proof of Concept**

---

## ✨ Features

- 📄 **PDF & JSON ingestion** — Upload documents directly in the browser; text is extracted client-side using `pdfjs-dist`
- 🔍 **Retrieval-Augmented Generation (RAG)** — Documents are chunked, embedded, and stored as vectors in Supabase; relevant context is retrieved at query time
- 🤖 **100% Local LLM** — Answers are generated by [Ollama](https://ollama.ai) running `gemma:2b` on your machine — no data is sent to external APIs
- 🧠 **Local embeddings** — Vector embeddings are generated via `nomic-embed-text` (768-dim) through Ollama, keeping all data private
- 🗄️ **Supabase vector store** — Uses `pgvector` for cosine-similarity search across large documents; falls back to full-context retrieval for small documents
- ⚡ **Vite + React + TypeScript** — Fast development experience with a clean, responsive UI built with Tailwind CSS and Framer Motion

```

│                     Browser (React App)                   │
│                                                           │

│                                                           │
│  User Query ──► Ollama Embeddings ──► Supabase RPC        │

└───────────────────────────────────────────────────────────┘
         │                              │
   Ollama (local)              Supabase (self-hosted /
   gemma:2b + nomic-embed      cloud with pgvector)
```

## 🚀 Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/encyclopedia.git
cd encyclopedia
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up Ollama models

Make sure Ollama is running, then pull the required models:

```bash
# Language model for generating answers
ollama pull gemma:2b

# Embedding model for vector search
ollama pull nomic-embed-text
```

Verify Ollama is running at `http://localhost:11434`.

### 4. Set up Supabase

1. Create a new project at [supabase.com](https://supabase.com)
2. In your Supabase dashboard, go to **SQL Editor** and run the migration script:

```sql
-- File: supabase/migrations/20260312000000_init_vector_db.sql
```

Copy the contents of the file above and execute it. This will:
- Enable the `pgvector` extension
- Create the `document_chunks` table (with 768-dim embedding column)
- Set up Row Level Security policies
- Create the `match_document_chunks` RPC function for similarity search

### 5. Configure environment variables

Create a `.env` file in the project root (copy from `.env.example` if available):

```env
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

> ⚠️ **Never commit your `.env` file.** It is already listed in `.gitignore`.

Get these values from your Supabase project: **Settings → API**.

### 6. Start the development server

```bash
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser.

## 💬 How to Use

1. **Upload a document** — Click "Upload Document" in the left sidebar and select a PDF or JSON file
2. **Wait for indexing** — The app extracts text, generates embeddings via Ollama, and stores them in Supabase. Progress is shown in real time
3. **Ask questions** — Type your question in the chat input and press Send
4. The app retrieves the most relevant chunks from your document and sends them as context to the local LLM
5. Answers are based **only** on the uploaded document — the model will not hallucinate or use outside knowledge

## 🔒 Privacy & Security

- **No external LLM calls** — All inference is performed locally through Ollama
- **No telemetry** — The app does not send any analytics or usage data
- **Your Supabase instance** — Vectors are stored in your own database, not a shared service
- **`.env` is gitignored** — Supabase credentials are never committed to the repository
